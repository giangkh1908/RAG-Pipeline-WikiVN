# Plan: Parallel batch embedding + parallel document processing

## Mục tiêu

Phá vỡ pipeline đang chạy tuần tự:

```
doc1 → doc2 → doc3 → ... → gom 250 chunk → embed → upsert → docN+1 ...
```

Thành pipeline **song song 2 tầng**:

1. **Tầng document** (multiprocessing) — normalize + clean + chunk nhiều document cùng lúc
2. **Tầng embedding** (ThreadPoolExecutor) — gửi nhiều sub-batch OpenRouter song song trong 1 lần flush

## Các thay đổi cụ thể

### 1. Thêm `ParallelEmbedder` — wrapper song song cho embedder

**File: `src/rag_pipeline/indexing/embedder.py`** (thêm class, không sửa existing code)

```python
@dataclass(slots=True)
class ParallelEmbedder:
    """Wrap một Embedder, chia batch lớn thành sub-batch nhỏ gửi song song."""
    inner: Embedder
    max_workers: int = 4
    sub_batch_size: int = 64

    def embed_texts(self, texts: list[str]) -> list[list[float]]:
        sub_batches = [texts[i:i+self.sub_batch_size]
                       for i in range(0, len(texts), self.sub_batch_size)]
        if len(sub_batches) <= 1:
            return self.inner.embed_texts(texts)

        from concurrent.futures import ThreadPoolExecutor, as_completed
        with ThreadPoolExecutor(max_workers=self.max_workers) as pool:
            futures = {pool.submit(self.inner.embed_texts, sb): i
                       for i, sb in enumerate(sub_batches)}
            results = [None] * len(sub_batches)
            for f in as_completed(futures):
                idx = futures[f]
                results[idx] = f.result()
        return [v for r in results for v in r]  # flatten, giữ thứ tự
```

**Tại sao**: Phần embedding là I/O-bound (HTTP POST), ThreadPoolExecutor giấu network latency. 
Mỗi sub-batch request chạy đồng thời, giảm total wall time từ `sum(latency_i)` xuống `max(latency_i)`.

### 3. Thêm `run_parallel` — document-level parallelism

**File: `src/rag_pipeline/pipelines/ingest_pipeline.py`** — thêm method mới

```python
def run_parallel(
    self,
    records: list[SourceRecord],
    processing_workers: int = 2,
) -> list[IndexingResult]:
    """Parallel version: xử lý documents song song qua multiprocessing."""
```

**Logic:**

```
Input: list[SourceRecord]
│
├── Bước 1: Chia records thành processing_workers phần
├── Bước 2: Dùng multiprocessing.Pool(processing_workers).map(
│       _process_records_chunk,
│       [(normalizer, cleaner, chunker, group) for group in groups]
│   )
│   Mỗi worker trả về list[(document, list[chunk])]
│
├── Bước 3: Gom chunks từ tất cả workers
│   for doc, chunks in all_results:
│       if checksum chưa seen:
│           pending.append((doc, chunks))
│           if pending >= embed_batch_size:
│               _flush_batch(pending, results)
│
└── Bước 4: Flush batch cuối
    if pending:
        _flush_batch(pending, results)
```

**Worker function (standalone ở module level để pickle được):**

```python
def _process_records_chunk(args):
    normalizer, cleaner, chunker, records = args
    output = []
    for record in records:
        doc = normalizer.normalize(record)
        doc.content = cleaner.clean(doc.content)
        chunks = chunker.chunk(doc)
        output.append((doc, chunks))
    return output
```

**Tại sao**: `SemanticChunker.encode()` dùng SentenceTransformer (PyTorch) — CPU-bound.
ThreadPoolExecutor bị GIL chặn, multiprocessing bypass GIL, tận dụng multiple cores.
